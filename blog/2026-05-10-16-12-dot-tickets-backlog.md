# Dot Tickets Backlog

Leanix now has a `.tickets` backlog compatible with the local dottickets tools.

The first tickets capture project-level maturity work rather than narrow coding
tasks:

- structured builders
- structured validation errors
- proof-backed package closure
- golden renderer tests
- GitHub Actions CI
- source trust modeling

This should make the next steps easier to browse across branches and worktrees
without turning every small commit into a ticket.
