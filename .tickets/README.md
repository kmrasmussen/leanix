# Leanix Tickets

This backlog captures durable project direction for Leanix.

Use tickets for semantic milestones, proof targets, renderer architecture,
Rust harness infrastructure, and design debt that should survive across
individual coding sessions. Do not add a ticket for every fixture, typo, or
narrow commit.

Each significant implementation slice should still be small, tested, blogged,
and committed independently. Tickets are the map; commits are the trail.

## Roadmap Wave

The active post-`TICKET-0025` roadmap wave is materialized as `TICKET-0026`
